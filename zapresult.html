Unit 3 Team Discussion: What is a Secure Programming Language?

 

You should read Chapter 2,6,7,8 of the course text (Pillai, 2017) and Cifuentes & Bierman (2019) and then answer the questions below, adding them as evidence to your e-portfolio.

 

    What factors determine whether a programming language is secure or not?

    Could Python be classed as a secure language? Justify your answer.

    Python would be a better language to create operating systems than C. Discuss.

 

Answers

 

1.    What factors determine whether a programming language is secure or not?

One of the criteria a programming language is considered secure is if it can prevent common memory related errors (e.g., buffer overflows, null pointer dereferences)

Another criterion is if it can be considered type safe; that is, it is a strongly typed language that helps prevent type errors that can lead to vulnerabilities.

Yet another criterion is how this language handles errors; how it prevents unexpected behavior that might be exploited.

If a given language uses secure standard libraries, that would reduce the needs for third-party libraries, which might introduce vulnerabilities.

 

2.    Could Python be classed as a secure language? Justify your answer.

Python can be described as a secure language:

Python is memory-safe and it handles memory management automatically, reducing the risk of memory-related vulnerabilities.

Python is dynamically typed, which can introduce some risks, but its strong typing system helps mitigate many issues.

Python¡¦s extensive standard library is well-maintained, but reliance on third-party packages can introduce vulnerabilities if not managed properly.

Python has a large, active community that regularly updates and patches security issues.

 

3.    Python would be a better language to create operating systems than C. Discuss.

No, Python is typically operating at a high level such as scripting, whereas C can operate in a low level, close to devices and hardware, which makes C ideal to write device drivers and system programming.
